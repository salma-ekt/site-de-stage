Présentation du projet de stage

Structure d’accueil : Sopra Steria

Service : NFS – Network Facing System

Contexte : Le service NFS développe et maintient des applications internes pour Orange Business, utilisées dans la gestion et le diagnostic des routeurs de leurs clients entreprises.

Planification

Objectif principal : Mettre en conformité l’interface web avec les exigences RGAA avant le 28 juin 2025.

Missions réalisées :

  • Mise en application des compétences acquises avec des missions concrètes (ex: corrections, petites fonctionnalités)
  • Travail sur des outils internes et la maintenance de l’application

Mission principale en cours :

  • Création d’une page affichant les résultats d’un audit RGAA
  • Ajout d’un lien dynamique dans le footer vers cette page
  • Traduction dynamique du libellé selon la langue

Planning

Semaine Description
Semaine 1 à 3 Formation obligatoire, apprentissage de Angular et Java
Semaine 4 à 5 Mise en application des compétences acquises avec des missions concrètes
Semaine 6 à 8 Projet principal : création et intégration de la page RGAA

Analyse du besoin

Le cœur du projet est la collecte de données sur les routeurs via IPCFM. Un audit RGAA a révélé la nécessité d’améliorer l’accessibilité de l’interface. L’objectif est de rendre ces informations accessibles à tous et conformes à la législation européenne.

Veille technologique et recommandations

Technologies utilisées : Angular, Bootstrap, SCSS, Cypress, Docker, Kubernetes, Quarkus, Java, Regex, Ansible, Python, Perl, Kafka, MongoDB, MariaDB, C, VirtualBox, ELK, GitLab CI.

Problème identifié : Utilisation de VirtualBox qui est lourd et peu pratique pour le développement quotidien.

Recommandation : Utiliser WSL2 (Windows Subsystem for Linux 2) à la place de VirtualBox pour une meilleure performance, une meilleure intégration avec Windows, et une expérience développeur optimisée.
